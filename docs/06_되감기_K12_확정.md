# 06. 되감기 평가 대상 확정 (K = 12)

**목적:** agent를 리콜 발표 이전 시점 데이터로 되돌려, 시그널을 스스로 발견하는지 정량 평가하기 위한 검증 대상 리콜 확정.
**후보 풀:** 현대·기아 SW/전장 캠페인 76건 (lookback_candidates.csv) 중 12개월 추이 확보 가능한 46건.

---

## 1. 선정 원칙

1. **유형 균형** — 잘 잡힐 것만 고르면 체리피킹. 스파이크형(A)·만성형(B)·조기급증형(C)을 섞어 성공과 실패가 모두 나오도록 구성
2. **제외 기준** — `insufficient_data=True` (2022년 이전 리콜 30건, 작업셋 시작 이전이라 추이 허수)
3. **검증 가능성** — 한국 리콜 매칭 사례(투싼 +8일, 쏘렌토 +10일)를 포함해 한·미 시차 기능과 연동 평가

## 2. 되감기에서 발견한 리콜의 3유형

- **유형 A (스파이크형):** 리콜 직전 1~3개월 불만 급증 → 급증 감지(파이프라인 1단계)가 잡아야 하는 대상
- **유형 B (만성형):** 불만 총량은 크지만 추이 평탄 → 건수 감지로는 불가, 서술문 내용 변화(LLM 구조화 분포)로만 포착 가능성. 실패 분석의 핵심 재료
- **유형 C (조기급증형):** 리콜 5~9개월 전 급증 후 소강 → "직전 3개월" 창만 보면 놓침. 감지 로직에 과거 급증 이력 반영 근거. 성공 시 조기경보 가치가 가장 큼

## 4. 평가 프로토콜 (초안)

- 각 캠페인의 접수일(report_date_iso)을 **컷오프**로, 그 이전 데이터만 agent에 제공
- 측정: ① 시그널 발견 여부 ② 발견 시점 vs 리콜 접수일 (선행 일수) ③ 지목한 부품·증상이 실제 리콜 사유와 일치하는지 ④ 리포트의 인용 정확성
- 결과 보고는 유형별 분리: "유형 A 재현율 x/5, C x/3, B x/4" — 유형 B 실패는 예상된 실패로서 원인 분석(건수 신호의 구조적 한계)과 함께 제시
- 유형 B 보조 실험: 건수 대신 **증상 구성 변화**(LLM 구조화 결과의 월별 분포)로 신호가 잡히는지 — 잡히면 "2축 감지"의 근거, 안 잡혀도 실패 분석

## 5. 알려진 한계 (기록)

- 월별 수치는 **차종 단위 매칭** → 같은 차종의 다른 결함 불만이 섞임 (예: EV9 세 캠페인이 추이 공유, 25V064/065는 동일 수치). 건수는 차종까지만 알고 "무슨 결함인지"는 서술문만이 안다 — LLM 필요성의 추가 논거이자, 평가 시 해석 주의점
- 판매량 미보정 — ratio(자기 대비 증가율) 중심으로 판단하는 이유
- ※확인 필요: 25V893000/25V874000(트레일러 히치)가 SW 후보에 포함된 사유 — Summary의 배선(wiring) 언급으로 추정. 키워드 필터 오탐이면 오류 사례로 기록
