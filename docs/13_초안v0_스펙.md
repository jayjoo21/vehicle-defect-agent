# 초안 v0 스펙 — 차량 결함 조사 AI Agent 웹 서비스

> 이 문서는 Claude Code에 그대로 전달하는 구현 명세입니다.
> 목표: 3화면(상황판 / 조사 채팅 / 내 차) + FastAPI 백엔드 + 목(mock) LLM 모드로 **키 없이 데모가 끝까지 도는** 초안.
> 이후 계속 수정·확장할 살아있는 초안이므로, 구조를 깨끗하게 유지하는 것이 완성도보다 우선.

## 0. 기술 스택 & 프로젝트 구조

- 프론트: **React + Vite + TypeScript + Tailwind + framer-motion + @react-three/fiber + @react-three/drei + recharts**
- 백엔드: **FastAPI (Python 3.11+)** — 기존 엔진 스크립트(scripts/)를 import해 재사용
- DB: **SQLite** (`data/app.db`) — 추후 Postgres/Supabase 전환 가능하게 SQL은 표준 문법만
- LLM: 어댑터 패턴 + 목 모드 (7절)

```
webapp/
  frontend/            # Vite React 앱
    src/
      pages/           # Dashboard.tsx, Chat.tsx, MyCar.tsx
      components/      # SignalCard, GapDumbbell, LifecycleTimeline, Heatmap, CarViewer, InvestigationTimeline ...
      lib/             # api.ts (fetch 래퍼), sse.ts, tokens.ts (디자인 토큰)
      mock/            # 목데이터 JSON (8절)
  backend/
    main.py            # FastAPI 앱
    routers/           # signals.py, vehicles.py, chat.py, reports.py
    engine/            # 기존 scripts/ 를 모듈화해 import
    llm/               # adapter.py (7절), mock_responses/
    db.py, models.py, seed.py   # 테이블 정의 + 목데이터 적재
  .env.example         # LLM_PROVIDER= / ANTHROPIC_API_KEY= / OPENAI_API_KEY=
```

## 1. 디자인 토큰 (tokens)

**철학: 테슬라의 여백 + 현대 네이비는 포인트로만 + 상태색이 유일한 주인공.**

```ts
colors: {
  bg:        '#FFFFFF',      // 페이지 배경 (기본은 라이트)
  bgSubtle:  '#F6F7F9',      // 카드 구분용 옅은 회색
  navy:      '#002C5F',      // 현대 네이비 — 헤더, 핵심 숫자, CTA, 차 실루엣 선에만
  navySoft:  '#E8EEF5',      // 네이비 틴트 (선택 상태 배경)
  ink:       '#111318',      // 본문 텍스트
  inkMuted:  '#6B7280',      // 보조 텍스트
  // 상태색 — 시선을 끄는 유일한 채도
  stateNew:      '#6B7280',  // 회색: 이력 없음/잠잠
  stateRising:   '#F59E0B',  // 노랑: 신고 증가
  stateActive:   '#DC2626',  // 빨강: 활성 시그널
  stateRecalled: '#2563EB',  // 파랑: 리콜 진행 중
  stateResolved: '#059669',  // 초록: 시정 완료
}
```

- 타이포: 시스템 산세리프(Pretendard 있으면 사용). **큰 숫자(32~48px, 네이비, weight 600) + 작은 라벨(12~13px, inkMuted)** 조합이 기본 패턴. 예: "**14건** / 신고 급증 · 최근 30일"
- 여백: 8pt 그리드. 카드 패딩 24px, 카드 간격 16~24px. 화면 좌우 최대폭 1200px 중앙 정렬
- 라운드 12px, 그림자 최소(`0 1px 3px rgba(0,0,0,.06)`), 테두리 `#E5E7EB` 1px
- **다크 예외 구역 딱 한 곳:** "내 차" 페이지의 3D 히어로 카드만 다크(#0B1220) — 150pt inspection 무드. 나머지 전부 라이트
- 도넛 차트 금지. 아이콘은 lucide-react, 1.5px stroke 통일

## 2. 전 화면 공통 원칙 (3원칙의 UI 구현)

1. **출처 배지:** 모든 사실 문장·수치 옆에 `[ODINO 11616811]` `[26V400000]` 형태의 회색 모노스페이스 배지. 클릭 시 원문 팝오버(목데이터에 원문 포함)
2. **"이력 없음"은 정상 화면:** 빈 상태 일러스트 없이 담백하게 — "현재 이 차종·증상으로 감지된 시그널이 없습니다. 데이터 기준일: {date}" + 그래도 볼 수 있는 것(해당 차종 월별 추이) 제시
3. **고지문 상시:** 채팅 답변 하단과 리포트 하단에 고정 — "본 정보는 NHTSA·국토부 공개 신고 및 리콜 기록 기반이며, 개별 차량의 진단이 아닙니다. 신고는 미검증 소비자 제보를 포함합니다."
4. **데이터 기준일 표시:** 헤더 우측에 "데이터 기준: 2026-07-06 05:00 KST" 항상 노출 (실시간인 척하지 않기)

## 3. 화면 1 — 상황판 (Dashboard) `/`

레이아웃 (위→아래):

1. **헤더:** 좌측 서비스명(가칭 "RECALL RADAR" — 팀에서 변경 가능, 네이비), 우측 데이터 기준일 + [내 차] [조사 채팅] 네비
2. **KPI 스트립 (4칸):** 감시 차종 78 / 활성 시그널 N / 이번 주 신규 알람 N / 미국 리콜·한국 미조치 N — 큰 숫자+라벨 패턴, 마운트 시 카운트업(0.8s)
3. **활성 시그널 카드 그리드 (핵심 영역):** 카드 = 차종명, 상태 뱃지(상태색), 최빈 증상 한 줄, 최근 30일 신고 수 + 스파크라인, "리포트 보기" 링크. 상태 필터 탭(전체/증가/활성/리콜중/재발). 카드 stagger 등장(60ms 간격)
4. **2열 섹션:**
   - 좌: **한·미 시차 덤벨 차트** (5절-A)
   - 우: **차종×월 히트맵** (5절-C)
5. **최근 리포트 리스트:** 제목/차종/생성일, 클릭 → 리포트 뷰(마크다운 렌더 + 출처 배지)

## 4. 화면 2 — 조사 채팅 (Chat) `/chat`

- 좌측 대화 영역 + 우측 고정 **조사 타임라인 패널** (모바일에서는 답변 위 접이식)
- 입력 예시 플레이스홀더: "내 차 EV6인데 계기판이 깜빡여요"
- **동작 흐름 (SSE 스트리밍):** 질문 전송 → 타임라인에 단계가 하나씩 실시간 추가 → 완료 후 답변 렌더
  - 타임라인 노드: 아이콘+제목+한 줄 결과. 예: `차종 인식 → KIA EV6` / `이력 조회 → 최근 90일 신고 12건` / `리콜 대조 → EV6 해당 리콜 없음` / `유사 증상 검색 → TUCSON 26V400000 발견 (동일 증상)` / `검수 → 통과 (출처 4건 확인)`
  - 각 노드 등장 애니메이션: fade+slide-up 200ms, 진행 중 노드는 점 3개 로딩
- **답변 구성(고정 템플릿):** ① 한 줄 결론 ② 이 차종의 신고 이력(수치) ③ 리콜 여부·한미 시차 ④ 원문 인용 1~3건(출처 배지) ⑤ [상세 리포트 보기] 버튼 ⑥ 고지문
- **EV6 데모 시나리오(목 모드의 기본 응답):** "EV6 계기판 깜빡임" 질문 → 타임라인 5단계 → 답변: "EV6 자체의 계기판 관련 리콜은 현재 없습니다. 다만 동일 증상(주행 중 계기판 꺼짐)으로 투싼이 2026-07-02 한국 리콜을 시작했으며[26V400000], 같은 계기판 계통입니다. EV6에서는 최근 90일 유사 신고 N건이 확인됩니다[ODINO ...]."
- 데이터에 없는 질문(예: "브레이크 오일 언제 갈아요?") → 목 모드에서도 "이력 없음/범위 밖" 응답 시연 가능해야 함

## 5. 시각화 3종 스펙

**A. 한·미 시차 덤벨 차트** — 데이터: `kr_us_gap.csv` 상위/대표 8건. 가로축=날짜, 행=캠페인. 미국 접수●(네이비)——선——●한국 발표(빨강 계열), 선 길이=시차, 우측에 "+N일" 라벨. 싼타페 +152일 행은 자동 강조. draw-in 애니메이션(선이 좌→우로 그어짐, 600ms stagger). 미국만 있고 한국이 없는 건은 점선+"한국 미조치" 라벨 → 이 행들이 KPI "미조치 N건"과 연결

**B. 시그널 생애 타임라인** — 데이터: EV9 24V757000 실측. X=월(2024-01~2025-03), Y=월 신고 수 면적 차트 + 배경에 상태 밴드(회색 신규→노랑 증가→빨강 활성→파랑 리콜(2024-10-10 세로선+캠페인 배지)→초록 잠잠). 리포트 뷰와 "내 차" 상세에서 재사용

**C. 차종×월 히트맵** — 데이터: `b1_signals.csv` 원천 집계. 행=차종(신고 많은 상위 20), 열=최근 24개월, 셀 색=신고 수(bgSubtle→네이비 스케일), **알람 발화 셀은 빨강 2px 테두리**. 호버 툴팁: 차종/월/건수/발화 여부. 셀 클릭 → 해당 차종 상세

## 6. 화면 3 — 내 차 (MyCar) `/my-car`

1. **등록 (첫 방문):** 2단계만 — ① 차종 선택(검색 가능한 그리드, 현대·기아 주요 27종) ② 연식 선택. localStorage 저장(로그인 없음). 완료 시 히어로로 전환
2. **3D 히어로 (다크 카드):** R3F Canvas. 범용 저폴리곤 EV SUV glTF 1개(세단 선택 시 세단 모델, 없으면 SUV 공용) + 차종명·연식 텍스트 오버레이. 자동 미회전(초당 ~6°), 드래그 회전, 줌 비활성
   - **핫스팟 6개** (3D 앵커 + drei `Html`): 계기판(대시보드 위치) / 인포테인먼트(센터) / ICCU·충전제어(후방 하부) / 구동배터리(바닥) / ADAS 카메라(전면 상단) / 제동 SW(전륜 안쪽)
   - 핫스팟 = 원형 점, **색은 해당 차종×도메인의 실제 시그널 상태색**. 호버 시 라벨+미니 요약, 클릭 시 하단 상세 카드로 스크롤
   - 폴백: WebGL 불가/로딩 3초 초과 시 차 측면 일러스트 SVG + 동일 핫스팟 배치 (기능 동일 보장)
   - glTF 출처: CC0 저폴리 모델 (Poly Pizza "low poly car" 계열) → `frontend/public/models/`에 포함, DRACO 압축
3. **도메인 상세 카드 (핫스팟 클릭 시):** 도메인명, 상태, 이 차종의 관련 신고 수·추이 미니차트, 관련 리콜(있으면 캠페인 배지+한미 시차), 대표 원문 인용 1건, [이 증상 조사하기] → 채팅으로 질문 프리필
4. **등록 완료 첫 진입 연출:** 핫스팟이 도메인 순서대로 200ms 간격 점등 → 상태 요약 토스트 "IONIQ 5 (2023): 활성 시그널 1건, 리콜 진행 1건"

## 8. LLM 어댑터 & 목 모드

```python
# backend/llm/adapter.py
class LLM:
    def call(self, role: str, messages: list, **kw):
        # role: "structurize" | "investigate" | "judge" | "answer"
        provider = os.getenv("LLM_PROVIDER", "mock")
        # provider별 모델 매핑 (설정으로만 관리, 하드코딩 금지):
        # anthropic: structurize=haiku급, investigate/answer=sonnet급, judge=OPENAI 쪽
        # openai:    structurize=mini급,  investigate/answer=4o급,    judge=ANTHROPIC 쪽
        # mock:      llm/mock_responses/{role}/*.json 에서 시나리오 매칭 반환
```
- `.env`에 키가 없으면 자동으로 mock — **에러가 아니라 정상 데모 모드**
- mock_responses에는 최소 3시나리오: EV6 계기판(하이라이트) / IONIQ 5 충전(이력 있음) / 범위 밖 질문(이력 없음 응답)
- Judge는 investigate와 **다른 provider**로 라우팅되는 구조를 어댑터에 명시 (키 하나만 있으면 같은 provider로 폴백 + 로그 경고)

## 9. 목데이터 — 반드시 실측 기반

`data/processed/`의 실제 산출물을 seed로 변환해 사용 (지어내지 말 것):
- `b1_signals.csv` → signals·heatmap (알람 67건 실물)
- `report_24V757000.md` → reports 1번 (EV9 견본)
- `kr_us_gap.csv` → gap (투싼 +8일, 싼타페 +152일 포함 27건)
- `llm_struct_test_results.jsonl` → complaints 일부 (원문 인용 팝오버용)
- EV6 시나리오용: 투싼 26V400000 리콜 레코드 + EV6 유사 신고(실제 확인한 2026-07-04 건 기반) 수동 seed
- 도메인 상태 매핑: IONIQ 5=충전제어 활성 이력, EV9=계기판 리콜(파랑), TUCSON=계기판 리콜중 — 나머지 조합은 실데이터 없으면 회색(이력 없음)

## 10. 애니메이션 명세 (framer-motion)

- KPI 카운트업 0.8s / 카드 stagger 60ms / 타임라인 노드 fade+slide 200ms / 덤벨 draw-in 600ms stagger / 핫스팟 순차 점등 200ms / 상태 변경 펄스 1회(무한 반복 금지)
- 전부 `useReducedMotion()` 체크 — reduced면 즉시 표시

## 11. 구현 순서 (이 순서대로 커밋)

1. 스캐폴드: 프론트+백엔드 뼈대, 토큰, 라우팅, .env.example
