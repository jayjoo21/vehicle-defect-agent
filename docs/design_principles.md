# 2. 원칙 전문 (참조용)

## 접근성 (#1)
색상이 유일한 정보 채널이 되면 안 됨(색각이상 8%). 텍스트 대비 WCAG AA(4.5:1), 데이터 마크는 배경 대비 3:1. 차트에 대체 텍스트(유형+내용+핵심 요점). 호버 전용 인터랙션 금지(탭/클릭 병행). 초당 3회 이상 깜빡임 금지.

## 주석 디자인 (#2)
주석 없는 차트는 질문, 있는 차트는 답변. 가장 중요한 1~3개 특징(피크·이상치·단절)에만 주석. 범례 대신 직접 레이블링. 의미 있는 기준선(목표·평균) 제공. 주석은 데이터보다 가볍게(작은 텍스트·옅은 색), 문장이 아닌 구로.

## 보조로서의 색상 (#3)
주요 데이터는 위치·길이로, 색상은 강조·보조로. 회색+강조 패턴(전부 회색, 집중할 1~2개만 색). 팔레트 최대 5~7색. 그레이스케일 변환 시 인사이트가 사라지면 색 과의존.

## 색채 이론 (#4)
범주형=구별 색조(≤8), 연속형=단일 색조 그라데이션, 발산형=중립 중간점의 이중 색조. 무지개 팔레트 금지(viridis 등 지각 균일 팔레트). 빨강-초록 단독 쌍 금지. 큰 영역 저채도, 작은 강조만 고채도.

## 일관성 (#5)
같은 데이터 속성 = 같은 시각 처리. 색 매핑·축 규칙·폰트·인터랙션·시간 포맷을 전 화면 통일. 차트마다 같은 지표에 다른 색 할당 금지. 나란한 비교 차트의 y축 스케일 통일.

## 장식보다 맥락 (#6)
차트가 비어 보이면 장식이 아니라 맥락(참조선·비교 시리즈·주석·출처·불확실성)으로 채울 것. 3D·그라데이션 배경·클립아트·데이터 영역 워터마크 금지. 로고 대신 출처와 방법론.

## 데이터-잉크 비율 (#7)
데이터를 나타내지 않는 잉크(무거운 격자선·테두리·중복 레이블) 최소화. 격자선은 옅은 회색 또는 생략. 같은 값을 3중 인코딩(높이+색+레이블) 금지.

## 빈 상태 (#8)
빈 화면은 에러가 아니라 디자인 대상. 왜 비었고 어떻게 채워지는지 + 다음 행동 제시. NaN/undefined 노출 금지. 결측 구간을 매끈한 선으로 보간 금지(공백·파선으로 정직하게). 로딩=스켈레톤, 실패=재시도 버튼.

## 게슈탈트 원리 (#9)
근접성(간격으로 그룹), 유사성(같은 색·형태=같은 범주), 연속성(선으로 이으면 한 개체), 둘러싸기(프레임=강한 그룹화). 그룹 간/내 간격 차이로 계층 전달.

## 그리드와 정렬 (#10)
공통 기준선(0선·시작점) 정렬이 비교의 전제. 일관된 여백, 열 기반 레이아웃, 텍스트 좌측 정렬로 수직 앵커. 스몰 멀티플은 축 범위 공유 필수.

## 의도적 기본값 (#11)
첫 화면이 가장 중요한 질문에 바로 답해야. 기본 정렬=값/위험도순(알파벳순 금지), 기본 필터=가장 유용한 조건, 기본 뷰에서 핵심 인사이트가 이미 강조된 상태. 제목도 해석("3분기 매출 정점")으로.

## 절제 (#12)
하나의 차트, 하나의 메시지. 시리즈 4~5개 이상이면 분리. 이중 y축 금지. 단순하게 시작하고 상세는 툴팁·클릭으로. 모든 지표를 한 화면에 욱여넣는 싱크대 대시보드 금지.

## 선택지 제한 (#13)
간격(8px 배수)·타입(12/14/16/20/28/40...)·선굵기·불투명도를 유한한 스케일로 정의하고 그 안에서만 선택. 매직 넘버 대신 명명된 토큰. "이 차트만 특별하니까" 오버라이드 금지.

## 점진적 공개 (#14)
개요 먼저(5초 안에 큰 그림) → 확대·필터 → 세부는 요청 시(호버/클릭). 첫 로드에 전부 쏟아내기 금지. 드릴다운에는 돌아갈 길(뒤로/브레드크럼). 툴팁이 유일한 데이터 접근 수단이면 안 됨.

## 반응형 (#15)
축소가 아니라 재설계(리플로우→리사이즈→리디자인). 모바일: 탭 툴팁, 터치 타겟 44px, 측면 범례 금지. [v0에서는 부채로 기록]

## 스몰 멀티플 (#16)
같은 구조·축·스케일의 작은 차트 그리드. 패널별 다른 y축 스케일은 치명적 오류. 논리적 배열 순서(시간·순위). 6~50개 범주 비교에 최적.

## 스토리텔링 구조 (#17)
제목은 헤드라인(핵심 인사이트 문장). 맥락 설정 → 데이터 → 함의의 아크. "그래서 뭐?" 테스트를 통과 못 하는 차트는 제거. 시간 범위 자르기 등으로 거짓 인과 암시 금지.

## 타이포그래피 위계 (#18)
3~4단계면 충분: 제목(볼드) > 부제·주석 > 축 레이블 > 출처(극소). 제목은 본문의 1.5~2배. 폰트는 1종(최대 2종). 세로 회전 레이블·전부 대문자·10px 미만 금지.

## 시각적 위계 (#19)
1차=데이터 마크·핵심 주석, 2차=제목·레이블, 3차=격자선·출처. 크기 외에 굵기·불투명도로도 위계. 숨기지 말고 약화. squint 테스트: 눈 찡그렸을 때 데이터만 보여야 함.

## 여백 (#20)
여백은 구조 요소. 막대 간격은 막대 너비의 50~100%. 요소가 테두리에 닿거나 잘리면 안 됨. 빈 공간을 장식으로 채우지 말 것. 넉넉한 여백 = 권위와 개방감.
